Manuale Layout CSS

Capire cosa usare e quando usarlo.

1. Display: Block vs Inline vs Inline-Block

Ogni elemento HTML nasce con un comportamento predefinito.

A. Display: Block (Il Mattone)

Occupa tutta la larghezza disponibile. Va a capo da solo.

Sono un DIV (Block)
Vado a capo anche se c'è spazio 
div { display: block; width: 100%; }

B. Display: Inline (Il Testo)

Occupa solo lo spazio necessario. Non accetta width/height e margin verticali.

Sono uno SPAN (Inline) Sto vicino al mio amico 
span { display: inline; /* width e height vengono ignorati! */ }

C. Display: Inline-Block (L'Ibrido)

Sta in riga come il testo, ma puoi dargli dimensioni come un blocco.

Inline-Block 1
Inline-Block 2
.bottone { display: inline-block; width: 150px; height: 50px; }

2. Il Box Model (Margini vs Padding)

Padding: Spazio INTERNO (tra il testo e il bordo). È come l'imbottitura di un cappotto.
Margin: Spazio ESTERNO (tra questo elemento e gli altri). È come il tuo spazio personale in fila.

Il Codice CSS dietro le quinte: 
.scatola {
  width: 200px;
  padding: 20px;   /* Spazio interno (verde) */
  border: 5px solid black; /* Bordo (grigio) */
  margin: 20px;    /* Spazio esterno (giallo) */
}

3. Unità di Misura: Quale usare?

Non usare sempre i pixel. Il web è fluido.

200px (Fisso, non cambia mai)
50% (Metà del genitore)
50vw (Metà della finestra browser)
Quando usarli:
  • px: Per bordi o dettagli molto piccoli e fissi.
  • %: Per layout fluidi (es. colonne).
  • rem: Per i font (rispetta le impostazioni del browser utente).
  • vh/vw: Per sezioni a tutto schermo (es. Hero Section).

4. Flexbox (display: flex)

Usa Flexbox quando devi allineare elementi in UNA direzione (o riga o colonna).

Caso d'uso A: La Navbar (Space Between)

Voglio il logo a sinistra e il menu a destra.

Logo
  • Home
  • About
  • Contact
Struttura HTML: 
<nav class="navbar">
  <div>Logo</div>
  <ul>...</ul>
</nav>
CSS Fondamentale: 
.navbar {
  display: flex;             /* 1. Attiva la magia */
  justify-content: space-between; /* 2. Spinge gli elementi agli estremi opposti */
  align-items: center;       /* 3. Allinea verticalmente al centro */
}

Caso d'uso B: Centrare tutto (Il Santo Graal)

Voglio un elemento perfettamente al centro.

Sono al centro!
CSS per centrare tutto: 
.contenitore {
  display: flex;
  justify-content: center; /* Centra orizzontalmente (asse X) */
  align-items: center;     /* Centra verticalmente (asse Y) */
}

Caso d'uso C: Griglia Flessibile (Wrap)

5. CSS Grid (display: grid)

Usa Grid quando devi gestire SIA righe CHE colonne contemporaneamente (Layout 2D).

Caso d'uso: Layout di Pagina "Bento Box"

Header (100%)
Sidebar (1fr)
Main Content (3fr)
Footer (100%)
CSS Griglia (2 Colonne): 
.griglia {
  display: grid;
  /* Crea 2 colonne: la prima è 1 frazione, la seconda è 3 volte più grande */
  grid-template-columns: 1fr 3fr;
  gap: 10px; /* Spazio tra le celle */
}

Caso d'uso Avanzato: Galleria Responsive Automatica

Questa è la regola più potente di Grid. Crea colonne automatiche senza Media Queries.

Il trucco magico: 
.galleria {
  display: grid;
  /* "Riempi lo spazio con colonne larghe almeno 100px, ma se c'è spazio allargale (1fr)" */
  grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
  gap: 10px;
}

7. Media Queries (Adattarsi al Mobile)

Come cambiare il layout quando lo schermo diventa piccolo.

Esempio classico: Da 3 colonne a 1 colonna 
/* 1. Stile Base (Desktop First o Mobile First) */
.colonne {
  display: flex;
  flex-direction: row; /* Affiancate su PC */
}

/* 2. Quando lo schermo è più piccolo di 768px (Tablet/Mobile) */
@media (max-width: 768px) {
  .colonne {
    flex-direction: column; /* Una sotto l'altra su Mobile */
  }
}

8. Pattern di Layout Comuni (Copia e Incolla)

Ecco i layout che userai nel 90% dei siti web, con i tag corretti.

A. Il Blog Classico (Header + Main + Sidebar + Footer)

Ideale per articoli, notizie o siti aziendali semplici.

<header>
<main> (Contenuto)
<aside> (Sidebar)
<footer>
Struttura HTML Semantica: 
<body>
  <header>Logo e Menu</header>

  <div class="container-flex">
    <main>
       <article>
          <h1>Titolo Articolo</h1>
          <p>Testo...</p>
       </article>
    </main>

    <aside>Pubblicità, Link, Autore...</aside>
  </div>

  <footer>Copyright</footer>
</body>
CSS (Flexbox): 
.container-flex {
  display: flex;
  flex-wrap: wrap; /* Importante per il mobile! */
  gap: 20px;
}

main {
  flex: 3; /* Occupa il 75% dello spazio (3 parti su 4) */
}

aside {
  flex: 1; /* Occupa il 25% dello spazio (1 parte su 4) */
}

/* Su mobile, manda la sidebar sotto */
@media (max-width: 768px) {
  main, aside { flex: 100%; }
}

B. La Dashboard (Sidebar Fissa + Header + Content)

Ideale per pannelli di amministrazione o app web.

<nav>
<header>
<main>
<footer>
Struttura HTML Semantica: 
<div class="admin-grid">
  <nav class="sidebar">Menu Laterale</nav>
  <header class="topbar">Utente e Ricerca</header>
  <main class="content">Grafici e Tabelle</main>
  <footer class="bottom">Info Legali</footer>
</div>
CSS (Grid): 
.admin-grid {
  display: grid;
  /* Colonna 1: 250px fissa, Colonna 2: tutto il resto */
  grid-template-columns: 250px 1fr;
  /* Riga 1: Header, Riga 2: Main (auto), Riga 3: Footer */
  grid-template-rows: 60px 1fr auto;
  min-height: 100vh;
}

.sidebar {
  grid-row: 1 / -1; /* Occupa dalla riga 1 all'ultima (tutta l'altezza) */
  background: #333; color: white;
}

C. Landing Page (Hero + Features)

Il layout standard per vendere un prodotto o servizio.

<header class="hero">
<article>
<article>
<article>
Struttura HTML Semantica: 
<header class="hero">
  <h1>Titolo Grande</h1>
  <button>Clicca qui</button>
</header>

<section class="features">
  <article class="card">Icona + Testo</article>
  <article class="card">Icona + Testo</article>
  <article class="card">Icona + Testo</article>
</section>
CSS (Grid Auto-Fit): 
.hero {
  height: 80vh; /* 80% dell'altezza schermo */
  display: flex; justify-content: center; align-items: center;
}

.features {
  display: grid;
  /* Crea colonne automatiche larghe almeno 250px */
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px; padding: 40px;
}

E. Sticky Footer (Footer sempre in basso)

Il problema classico: hai poco contenuto e il footer sale a metà pagina. Ecco come fissarlo giù.

Header
Poco contenuto...
(Ma il footer resta giù!)
Footer
CSS (Flexbox Column): 
body {
  min-height: 100vh; /* 1. Il body deve essere alto ALMENO quanto lo schermo */
  display: flex;
  flex-direction: column; /* 2. Impila gli elementi verticalmente */
}

main {
  flex: 1; /* 3. "Prenditi tutto lo spazio vuoto disponibile" */
}
/* Risultato: Il main si allunga e spinge il footer in fondo. */
